# AI開発 公開前セルフチェックキット 無料版

AIで作った個人開発プロダクトを公開する前に、最低限確認したい項目だけをまとめた無料版です。

Claude Code、Cursor、Windsurf、Codex、ChatGPTなど、特定のAI開発ツールに依存せず使えます。

## 公開直前に起きがちなやらかし

AIで作ったものは、動いているように見えても公開直前に事故りやすいです。

- `.env` やAPIキーを混ぜたままGitHubに出す
- READMEにまだ実装していない機能を書いてしまう
- AIが作った販売文が実際より強くなっている
- BOOTHの商品説明にサポート範囲や注意書きを書き忘れる
- Zennやnoteで「できること」を盛りすぎる
- 主要導線を最後まで試さずに公開する

この無料版は、まず10分で最低限だけ見るためのチェックリストです。

## 10分セルフチェック

### 基本動作

- [ ] 初回アクセス時に主要画面が表示される
- [ ] 一番大事な操作を最初から最後まで実行できる
- [ ] 空文字、日本語、長文を入力しても壊れない
- [ ] エラー時に画面が真っ白にならない
- [ ] スマホ幅でも最低限使える

### 利用者目線

- [ ] 最初の画面で「何ができるものか」が分かる
- [ ] 使い始めるボタンや導線が迷わない
- [ ] 成功時に結果が分かる
- [ ] 失敗時に次の行動が分かる
- [ ] まだ使えない機能を、使えるように見せていない

### AI生成物の確認

- [ ] AIが作った文章に過剰な断定がない
- [ ] 実装していない機能を説明していない
- [ ] 架空の実績、架空の導入事例、架空の保証がない
- [ ] READMEや商品説明と実際の機能が一致している

### 公開リスク

- [ ] APIキー、トークン、パスワードを公開していない
- [ ] `.env` や秘密情報ファイルを公開していない
- [ ] 個人情報や実在ユーザーのデータを含めていない
- [ ] 使用している素材やライブラリの利用条件を確認した
- [ ] 注意書き、免責、サポート範囲を最低限書いた

## 無料版の公開判断

以下のどれに当てはまるかで判断してください。

### 公開してよい

- 主要導線が動く
- 秘密情報が含まれていない
- 実際の機能と説明が一致している
- できないことを注意書きで説明できている

### 制限付きで公開

- 一部の環境で不安がある
- まだ荒いが、利用者に大きな不利益は出にくい
- 商品ページやREADMEに制限を書けば誤解を減らせる

### まだ公開しない

- 秘密情報が混ざっている可能性がある
- 主要機能が途中で止まる
- 説明と実際の機能が大きく違う
- 利用者のデータを壊す、漏らす、誤解させる可能性がある

## 有料版が必要になりやすい人

無料版は最低限の10分チェックです。

以下に当てはまる場合は、有料版の方が使いやすいです。

- BOOTH、GitHub、Zenn向けにもう少し細かく確認したい
- AIにそのまま貼れるレビュー用プロンプトが欲しい
- 「公開してよい / まだ公開しない」を記録したい
- 販売文やREADMEの誇張チェックを毎回やりたい
- 個人開発プロダクトを継続的に公開する予定がある

## 有料版に含まれるもの

- 公開前30分チェック
- Webアプリ、GitHub、BOOTH、Zenn向けの深掘りチェック
- AIに依頼するレビュー・修正プロンプト集
- 公開判定シート
- 注意書きテンプレート
- BOOTH販売前の不安を減らす確認項目
